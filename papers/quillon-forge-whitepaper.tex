\section{Executive Summary}
% ============================================================================

The \textbf{Quillon Forge} is a purpose-built mining appliance for the Q-NarwhalKnight post-quantum blockchain. Unlike Bitcoin ASICs that optimize for SHA-256 hash grinding, the Forge is architected around the computational demands of \textbf{post-quantum cryptography} (Dilithium5, Kyber1024), \textbf{Verifiable Delay Functions} (Genus-2 hyperelliptic VDFs), and \textbf{AI inference} (Proof of Inference rewards).

Each Forge unit ships as a \textbf{plug-and-play node} --- power it on, connect Ethernet, and it begins mining within 60 seconds. No Linux expertise required. The on-board firmware handles blockchain synchronization, peer discovery, key generation, and mining optimization automatically.

\section{Why CPU-First Mining}
% ============================================================================

Q-NarwhalKnight's consensus differs fundamentally from Bitcoin and Ethereum:

\begin{enumerate}
    \item \textbf{Post-Quantum Signatures} (Dilithium5): Block validation requires lattice-based signature verification --- CPU-bound, AVX-512 accelerated, not GPU-parallelizable.

    \item \textbf{Verifiable Delay Functions} (Genus-2 VDF): Anchor election uses sequential Jacobian doubling on hyperelliptic curves --- \textit{inherently sequential}, benefits from high single-core clock speed.

    \item \textbf{Proof of Inference}: AI model inference earns bonus QUG rewards --- benefits from both CPU (llama.cpp) and GPU (CUDA acceleration).

    \item \textbf{DAG-Knight Consensus}: Block production requires parallel processing of DAG vertices with Bracha's reliable broadcast --- scales linearly with core count.
\end{enumerate}

\subsection{Automatic Optimization}

The Q-NarwhalKnight node binary (\texttt{q-api-server}) includes automatic hardware detection and optimization (v5.1.0):

\begin{itemize}[noitemsep]
    \item \textbf{CPU cores}: Auto-detects physical cores via \texttt{num\_cpus::get\_physical()}, spawns one mining thread per core with CPU affinity pinning.
    \item \textbf{AVX-512}: Runtime detection of AVX-512F/DQ/BW/VL. Batch signature verification uses 512-bit SIMD when available.
    \item \textbf{NUMA}: Detects dual-socket topology via \texttt{/sys/devices/system/node/}, pins threads to local NUMA nodes to minimize cross-socket memory latency.
    \item \textbf{RocksDB}: Auto-scales background jobs: $\lfloor\text{cores}/16\rfloor$ background threads (clamped 2--16), $\lfloor\text{cores}/32\rfloor$ compaction threads.
    \item \textbf{AI inference}: \texttt{LlamaCppEngine} scales concurrent requests: $\lfloor\text{cores}/32\rfloor$ (clamped 2--16), overridable via \texttt{LLAMA\_MAX\_CONCURRENT}.
    \item \textbf{Batch crypto}: Signature batch size scales to $\min(8 \times \text{cores}, 4096)$, hash batch to $\min(4 \times \text{cores}, 2048)$.
\end{itemize}

\section{Hardware Attestation \& Security}
% ============================================================================

Each Forge unit contains a hardware attestation subsystem:

\begin{itemize}[noitemsep]
    \item \textbf{TPM 2.0}: Stores libp2p identity key and mining wallet seed phrase in tamper-resistant hardware.
    \item \textbf{Machine ID}: Unique 256-bit identifier burned into firmware at factory, recorded on-chain at fulfillment.
    \item \textbf{Attestation Public Key}: Ed25519 key pair generated in TPM, public key registered on-chain. Used to sign mining submissions, proving they originate from genuine Forge hardware.
    \item \textbf{Secure Boot}: UEFI Secure Boot with Quillon signing key. Prevents unauthorized firmware modifications.
    \item \textbf{Encrypted Storage}: NVMe drives encrypted with AES-256-XTS, key derived from TPM + user password.
\end{itemize}

\section{RWA Token Lifecycle}
% ============================================================================

\begin{enumerate}
    \item \textbf{Mint}: 500 \texttt{FORGE} tokens minted to \texttt{BANK\_MASTER\_ACCOUNT} at genesis.
    \item \textbf{Purchase}: Users acquire \texttt{FORGE} tokens on the Q-NarwhalKnight DEX or directly from Quillon.
    \item \textbf{Configure}: At redemption, user selects CPU (EPYC/Xeon), GPU (none/RTX/A100/L40), RAM (512/1024/2048\,GB), cooling, chassis color.
    \item \textbf{Burn}: Token burned on-chain, creates \texttt{ForgeRedemption} order with unique \texttt{FR-*} ID.
    \item \textbf{Assemble}: Status progresses: \texttt{pending} $\rightarrow$ \texttt{configured} $\rightarrow$ \texttt{assembling} $\rightarrow$ \texttt{testing} $\rightarrow$ \texttt{shipped} $\rightarrow$ \texttt{delivered}.
    \item \textbf{Activate}: On first boot, Forge registers its \texttt{machine\_id} and \texttt{attestation\_pubkey} on-chain.
    \item \textbf{Mine}: Forge begins mining QUG and earning Proof of Inference rewards.
\end{enumerate}
