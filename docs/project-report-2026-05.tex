\section{Executive Summary}

The Q-NarwhalKnight (QNK) network is \textbf{live, syncing, and ready for public release}
with v10.8.3 deployed across all four production servers (Beta, Gamma, Delta, Epsilon).
Three critical sync-related data integrity bugs — SYNC-002, SYNC-005, and SYNC-006 —
were identified and fixed during May 2026, preventing balance divergence and block
history loss on checkpoint-bootstrapped nodes.

\textbf{One open risk remains:} BAL-001 (BalanceRootV1 consensus rule) activates
$\approx$1.05\,million blocks from now ($\approx$Q3 2026). Pre-activation integrity
verification between Epsilon and fresh-sync nodes is mandatory before that height is reached.

Between March 1 and May 11, 2026, the project shipped \textbf{416 commits} across
28 crates, resolved 6 SYNC-class critical issues, and produced 78 technical review
documents. The period progressed through three major engineering themes:

\begin{enumerate}[leftmargin=*,itemsep=2pt]
  \item \textbf{Data Integrity Hardening} (SYNC-002, SYNC-006) — balance divergence
    and block history loss on warp-sync nodes; max-wins guard mandated in CLAUDE.md.
  \item \textbf{P2P Gap-Fill Reliability} (SYNC-005, v10.8.2/3) — root cause of
    30--120\,s block-range timeouts isolated and eliminated.
  \item \textbf{Infrastructure Hardening} — HA rolling deploy, multi-server glibc
    compatibility, Tor Browser WebRTC call support.
\end{enumerate}

\section{Technical Findings}

\subsection{SYNC-001 — Block History Loss on Warp-Sync Nodes}

When a fresh node starts with \texttt{local\_height < 100}, \texttt{turbo\_sync.rs}
calls \texttt{probe\_network\_gap()} which binary-searches peers to find the lowest
available block height — the checkpoint at $\approx$16.75M.
It sets \texttt{effective\_start\_height = gap\_floor} and downloads only blocks above
that point. Blocks 1--16,749,999 are never requested.
\texttt{current\_height\_atomic} reports the \emph{maximum} received height (17.5M)
rather than the contiguous stored height, so the node advertises full history it
cannot serve.

\textbf{Short-term mitigation:} \texttt{Q\_ARCHIVE\_NODE\_URL} proxy.
\textbf{Long-term fix:} \texttt{Q\_SKIP\_CHECKPOINT=1} forces
\texttt{effective\_start\_height = 1}; \texttt{NODE\_TYPE=archive|full|light} flag
prevents light nodes being selected for history by gap-fill.

\subsection{SYNC-002 / BAL-001 — Balance Divergence (62 wallets, 27,235 QUG)}

Three co-contributing bugs caused balance divergence on fresh checkpoint-bootstrapped
nodes vs Epsilon:

\begin{description}[leftmargin=0pt]
  \item[\textbf{Bug A — Transfer-skip threshold:}]
    When $>$5,000 blocks behind, turbo sync called
    \texttt{process\_block\_coinbase\_only\_tx()} — rewarding mining but skipping
    all transfer transactions across 17.5M blocks.
    The 62 missing wallets received funds only via transfers, never via mining.
    \emph{Fixed v10.7.1: threshold removed; all transactions always processed.}

  \item[\textbf{Bug B — Supply not persisted:}]
    Turbo sync wrote wallet balances to RocksDB but never persisted
    \texttt{total\_minted\_supply}; on restart it recomputed from the wrong wallet map.
    \emph{Fixed v10.7.1: supply summed and persisted after every batch commit.}

  \item[\textbf{Bug C — Post-sync reload omits supply:}]
    The \texttt{now\_synced \&\& !was\_synced} handler reloaded \texttt{wallet\_balances}
    but not \texttt{total\_minted\_supply}.
    \emph{Fixed: supply recomputed from reloaded balances in sync-complete handler.}
\end{description}

\subsection{SYNC-006 — Balance Replay Corruption (May 9 Incident)}

\texttt{replay\_post\_checkpoint\_balances()} ran on Epsilon (a genesis node with
correct balances) because \texttt{is\_checkpoint\_applied()} was not checked.
The replay overwrote high balances with lower checkpoint values
(confirmed case: one wallet 3,200\,QUG $\rightarrow$ 1,484\,QUG).

\textbf{Fix (CLAUDE.md Rules 1--3):} \texttt{save\_wallet\_balances} uses a
\emph{max-wins} guard (skip write if existing DB value is higher); replay gates on
\texttt{is\_checkpoint\_applied()} and never runs on genesis nodes
(\texttt{is\_genesis\_node()} guard).

\subsection{SYNC-005 — P2P Gap-Fill 30/120\,s Timeouts (v10.8.2 + v10.8.3)}

\paragraph{Server-side root cause (v10.8.2):}
The block-pack responder called \texttt{get\_qblocks\_range\_any\_format}
(O($n \times 3$) prefix scans; $>$60\,s for 200 blocks) \emph{before}
\texttt{get\_qblocks\_forward} (O(1) forward iterator seek, $<$50\,ms).
\emph{Fix: reorder — forward-seek first; slow scan only as last resort.}

\paragraph{Client-side root cause (v10.8.3):}
\texttt{RequestBlockRangeDirect} stores \texttt{internal\_tx} in
\texttt{pending\_block\_requests}.
When \texttt{OutboundFailure} fires (peer rejects immediately), the handler logged
and updated the sync retry queue but \emph{never removed the pending entry}.
The spawned awaiter task blocked on \texttt{internal\_rx.await} indefinitely;
\texttt{fill\_gap\_p2p} always hit the full 120\,s wall even for instant rejections.
\emph{Fix: drop \texttt{pending\_block\_requests} entry inside \texttt{OutboundFailure},
resolving the channel with \texttt{Err} within milliseconds.}

\subsection{Pre-Mortem --- BAL-001 Activation}

\textit{Assumption: It is Q3 2026. Balance divergence occurred at BAL-001
activation. What caused it?}

\begin{enumerate}[leftmargin=*,itemsep=2pt]
  \item A checkpoint node running an old binary (pre-v10.7.1) retained the
    transfer-skip threshold --- its balance root diverged at activation height.
  \item A node ran \texttt{replay\_post\_checkpoint\_balances()} on first boot
    after upgrading, overwriting correct balances without the max-wins guard.
  \item \texttt{total\_minted\_supply} was recomputed from a stale in-memory
    wallet map after a node restart, producing an incorrect balance root.
  \item The height alarm at 18,400,000 was never deployed; pre-activation
    verification was not triggered, allowing divergence to persist undetected.
\end{enumerate}

\textbf{Prevention:} Scenarios 1--3 are mitigated by SYNC-002/006 and
CLAUDE.md Rules 1--3. Scenario 4 is the remaining P0 action (height alarm +
mandatory pre-activation balance-root comparison).

\subsection{Blameless Post-Mortem --- May 9 Balance Replay Incident}

\textbf{Incident:} \texttt{replay\_post\_checkpoint\_balances()} executed on
Epsilon (a genesis node with correct balances) after a scheduled restart,
overwriting correct wallet balances with lower checkpoint values.
Confirmed case: one wallet 3,200\,QUG $\rightarrow$ 1,484\,QUG.

\begin{table}[H]
\centering
\small
\begin{tabularx}{\linewidth}{lX}
\toprule
\textbf{Phase} & \textbf{Event} \\
\midrule
\textbf{Detect} (T+0) &
  User reported balance reduction via Discord.
  Server Beta observed wallet balance inconsistent with Epsilon logs. \\
\rowcolor{rowgray}
\textbf{Mitigate} (T+20\,min) &
  Node restart aborted. RocksDB snapshot taken for forensics.
  Epsilon kept offline to preserve DB state. \\
\textbf{Diagnose} (T+45\,min) &
  Log analysis confirmed \texttt{replay\_post\_checkpoint\_balances} executed;
  \texttt{is\_checkpoint\_applied()} returned \texttt{false} (genesis node,
  correct), but the call-site lacked the guard entirely. \\
\rowcolor{rowgray}
\textbf{Resolve} (T+2\,h) &
  Balances restored from RocksDB snapshot; max-wins guard added to
  \texttt{save\_wallet\_balances}; \texttt{is\_checkpoint\_applied()} guard
  added at replay call-site; \texttt{is\_genesis\_node()} guard added;
  CLAUDE.md Rules 1--3 codified. \\
\bottomrule
\end{tabularx}
\caption{Blameless post-mortem timeline --- May 9 balance replay incident.}
\end{table}

\textbf{Contributing factors:}
\begin{itemize}[leftmargin=*,itemsep=1pt]
  \item \texttt{save\_wallet\_balances} performed a batch overwrite with no
    read-before-write monotonicity protection.
  \item Replay function had no genesis-node guard; only the checkpoint guard
    was considered necessary.
  \item No automated test exercised the replay path on a node where
    \texttt{is\_checkpoint\_applied()} returns \texttt{false}.
\end{itemize}

\textbf{Action items:}
\begin{enumerate}[leftmargin=*,itemsep=1pt]
  \item Add RocksDB merge operator enforcing max-wins at the storage layer (P1).
  \item Add integration test: run replay on a genesis-node fixture and assert
    balances are \emph{unchanged} (P1).
  \item CLAUDE.md code-review checklist entry referencing
    \texttt{save\_wallet\_balances} and replay paths (Done --- Rules 1--3).
\end{enumerate}

  channel the request was associated with.

  whenever balances are written, making supply-write an unavoidable invariant. \\
