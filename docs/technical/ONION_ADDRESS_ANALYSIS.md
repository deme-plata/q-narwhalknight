# Q-NarwhalKnight Onion Address Analysis

## 🚨 **Critical Discovery: Dual Addressing System**

### **The Problem:**
Q-NarwhalKnight is generating `.qnk.onion` addresses that **are NOT valid Tor onion addresses**. When tested in Tor Browser, they return:

```
Invalid onion site address
Den angivne onionsted-adresse er ugyldig. Tjek venligst at du har indtastet den korrekt.
```

### **Root Cause Analysis:**

#### **1. Dual Addressing Architecture**
Q-NarwhalKnight implements **two separate addressing systems**:

- **Real Tor Addresses**: Standard `.onion` format (56 chars + `.onion`)
- **Quantum Network IDs**: Custom `.qnk.onion` format (56 chars + `.qnk.onion`)

#### **2. Code Evidence:**

**Real Tor Integration (Expects Standard .onion):**
```rust
// crates/q-tor-client/src/tor_socks.rs:41
if !onion_address.ends_with(".onion") {
    return Err(anyhow::anyhow!("Invalid onion address: {}", onion_address));
}

// crates/q-tor-client/src/real_tor_client.rs:656
assert!(onion_address.ends_with(".onion"));
```

**Q-NarwhalKnight Network IDs (Uses .qnk.onion):**
```rust
// crates/q-zk-p2p/src/lib.rs:149
assert!(address.ends_with(".qnk.onion"));

// crates/q-bep44-discovery/src/real_tor.rs:407
assert!(onion_address.ends_with(".qnk.onion"));
```

### **3. Current System Behavior:**

**Generated Addresses:**
- `57yam736tish5d7vrg2d4vmevumwsawhyxzl3qywelq45mcut4xq.qnk.onion` ❌ (Invalid for Tor)
- `cdj4nbpvx2ldb5mwus2t4qkua4owwzbrw3fd26swfkyc2iha2azq.qnk.onion` ❌ (Invalid for Tor)

**What Should Happen:**
- Generate real `.onion` addresses for actual Tor connectivity
- Use `.qnk.onion` format only for internal Q-NarwhalKnight identification
