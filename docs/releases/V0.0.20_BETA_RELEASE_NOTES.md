# Q-NarwhalKnight v0.0.20-beta Release Notes

**Release Date**: October 26, 2025
**Status**: Beta Release
**Focus**: Automatic Block Production & DAG Continuity

---

## 🎯 What's New

### Automatic Time-Based Block Production (ENABLED BY DEFAULT)

**The Big Change**: Blocks now produce automatically every 15 seconds, even without mining solutions!

**Why This Matters**:
- **DAG Continuity**: The DAG now grows consistently, enabling proper multi-node consensus testing
- **No More Empty Heights**: Users no longer see `height: null` when checking node status
- **Easier Testing**: Developers can test block propagation without needing mining solutions
- **Better UX**: Blocks are visible immediately, making the system feel more responsive

**Technical Details**:
- Modified `should_produce_block()` in `block_producer.rs` line 97-104
- Changed from: `(time_elapsed && enough_solutions) || max_solutions_reached`
- Changed to: `time_elapsed || max_solutions_reached`
- Blocks can now be created with empty `mining_solutions` vec for DAG continuity

**Endpoint**: `POST /api/v1/trigger-block`

**Purpose**: Force immediate block production for testing

**Usage**:
```bash

**Response**:

  }

```

**Use Cases**:
- Testing block propagation in multi-node environments
- Debugging DAG consensus finality
- Demonstrating the system without waiting for 15-second intervals
- Development and integration testing

## 🐛 Bug Fixes

### Fixed: Blocks Not Producing Without Mining Solutions

**Issue**: In v0.0.19-beta, nodes showed `height: null` because `produce_block()` returned `None` when `pending_solutions.is_empty()`

**Root Cause**:
- Line 111-114 in `block_producer.rs`:
  ```rust
  if self.pending_solutions.is_empty() {
      debug!("⏭️  No pending solutions, skipping block production");
      return None;
  }
  ```

**Fix**: Allow blocks with empty solutions for DAG continuity:
```rust
let solutions: Vec<MiningSolution> = if solutions_count > 0 {
    self.pending_solutions.drain(0..solutions_count).collect()
} else {
    // No solutions available - create empty block for DAG continuity
    debug!("📦 Producing empty block for DAG continuity (no mining solutions)");
    vec![]
};
```

## 📊 Upgrade Instructions

### From v0.0.19-beta to v0.0.20-beta

1. **Stop existing node** (if running):
   ```bash
   pkill q-api-server
   ```

2. **Pull latest code**:
   ```bash
   git pull origin clean-branch
   ```

3. **Rebuild** (use 10-hour timeout for safety):
   ```bash
   timeout 36000 cargo build --release --bin q-api-server
   ```

4. **Restart node**:
   ```bash
   ./target/release/q-api-server --port 8080
   ```

5. **Verify blocks producing**:
   ```bash
   # Wait 15-30 seconds, then check:
   curl http://localhost:8080/api/v1/status | jq '.data.current_height'
   # Should see height > 0 and advancing
   ```

This release is **100% backward compatible**. Existing RocksDB data works without changes.
